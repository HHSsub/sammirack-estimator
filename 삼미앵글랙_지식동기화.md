## 0. 절대 규칙 (위반 불가)

> [!CAUTION]
> 아래 규칙은 AI/작업자가 **절대로 위반해서는 안 되는** 불변 규칙이다.

| # | 규칙 | 비고 |
|---|------|------|
| 1 | **이 문서의 전체 구조·양식은 임의 수정 절대 금지** | 섹션 추가만 허용, 기존 구조 변경 불가 |
| 2 | **어떤 작업이든 반드시 이 문서를 먼저 읽고, 현재 진행 상황을 파악한 뒤 작업할 것** | 예외 없음 |
| 3 | **어떤 작업을 했든 반드시 작업 이력(History)에 prepend 방식으로 최신 이력을 맨 위에 추가** | 디버깅 1건, 주석 1줄, 코드 1글자, 설정값 1개 변경 포함 |
| 4 | **작업계획을 세울 때 "지식동기화 문서 업데이트" 단계는 항상 필수 단계로 포함** | 생략 불가 |
| 5 | **Step 테이블의 "사용자승인" 컬럼은 AI가 절대 임의 수정 금지 (only human touch)** | AI는 "대기" 또는 "미승인" 상태로만 제안 가능 |

| 7 | **서버 설정(nginx, 경로, 포트 등) 변경 시 추론/추정 절대 금지. 반드시 §9의 정확한 설정값 사용** | 경로·파일·포트·설정 등 인프라 관련은 100% source 기반으로만 작성 |

## 1. 현재 메인 작업 목표

**하이랙 신규 색상(아이보리) 추가 및 색상/옵션/자재/가격 아키텍처 리팩토링**

### 1.1 아이보리 신규 색상 사양

| 항목 | 값 |
|------|-----|
| 랙 종류 | 하이랙 |
| 신규 색상 | 아이보리 |
| WxD (사이즈) | 60×108, 60×150, 60×200 |
| 하중 | 270kg, 450kg |
| 높이 (기둥) | 150, 200, 250 |

### 1.2 리팩토링 방향 (필수 달성 목표)

| 현재 문제점 | 목표 상태 |
|---|---|
| "메트그레이" 등 색상명 하드코딩 (정규식, 조건분기, 매핑테이블) | 데이터 드리븐 색상 관리 (색상 추가 시 코드 수정 0건) |
| `public/` 폴더의 JSON/CSV 정적 데이터 의존 | 중앙 데이터 레지스트리 기반 동적 로딩 |
| 색상/규격/랙종류 추가 시 여러 파일 수동 수정 필요 | 단일 데이터 정의로 모든 시스템 자동 반영 |
| `EXTRA_TO_BASE_INVENTORY_MAPPING` 등 매핑 테이블 수동 확장 | 규칙 기반 자동 매핑 또는 매핑 데이터 외부화 |
| `bomDisplayNameUtils.js`의 색상별 분기 하드코딩 | 색상 메타데이터 기반 자동 표시명 생성 |

## 8. 하이랙 규격 절대 규칙 (Strict Rules)

> [!DANGER]
> 아래 규격은 **물리적/재고 관리적**으로 고정된 사실이며, 절대 임의로 확장하거나 축소해서는 안 된다.

### 8.1 하이랙 기둥 유효 규격 (Wx Only - Depth 제거)

**규칙**: 하이랙 기둥 ID 및 데이터는 `Depth`를 포함하지 않는다. 오직 `Width(폭)`와 `Height(높이)`만 사용한다.
*   **표기법**: `사이즈{Width}x높이{Height}` (예: `사이즈60x높이200`)
*   **주의**: `60x108`, `60x150` 등 깊이가 다른 랙이라도, 기둥은 동일한 `60x` 규격을 사용한다.

| 색상 | 하중 | 유효 사이즈 (Width) | 비고 |
|------|------|-------------------|------|
| **메트그레이** | 270kg | **45x, 60x** | 높이: 150/200/250 |
| **메트그레이** | 450kg | **60x Only** | **45x 없음** |
| **블루+오렌지** | 270kg | **45x, 60x** | 높이: 150/200/250 |

## 9. 서버 인프라 설정 (절대 변경 금지 — Source of Truth)

> [!CAUTION]
> **이 섹션의 설정값은 실제 운영 서버의 정확한 설정이다.**
> - AI/작업자는 이 설정을 추론/추정하지 않고 반드시 이 섹션의 값을 그대로 사용해야 한다.
> - nginx 설정을 수정할 때는 반드시 아래 전체 설정을 base로 사용하고, 기존 블록을 절대 누락하지 않는다.
> - **2026-02-16 사고 기록**: AI가 nginx 캐시 설정 수정 시 `/api/` 프록시 블록을 누락하여 프론트엔드 전체 데이터 로딩 불가 장애 발생. 이로 인해 이 섹션 신설.

### 9.1 서버 정보

|------|-----|
| 서버 IP | `139.150.11.53` |
| OS | Rocky Linux |

### 9.2 nginx 설정 (정확한 전체 설정 — 복사 전용)

> 아래 명령어를 **그대로** 서버에서 실행하면 nginx 설정이 적용된다.
> **주의**: bash heredoc 안에서 `$`는 `\$`로 이스케이프해야 한다.

```bash
sudo bash -c 'cat > /etc/nginx/nginx.conf <<EOF
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log;
pid /run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    server {
        listen       80;
        server_name  _;
        root         /home/rocky/sammirack-app;
        index        index.html;

        # index.html 캐시 금지
        location = /index.html {
            add_header Cache-Control "no-store, no-cache, must-revalidate, proxy-revalidate, max-age=0";
            add_header Pragma "no-cache";
            add_header Expires "0";
            try_files \$uri /index.html;
        }

        # JSON/CSV 캐시 금지
        location ~* \.(json|csv)\$ {
            add_header Cache-Control "no-store, no-cache, must-revalidate, max-age=0";
            add_header Pragma "no-cache";
            try_files \$uri =404;
        }

        # 빌드 산출물 장기 캐시
        location /assets/ {
            add_header Cache-Control "public, max-age=31536000, immutable";
            try_files \$uri =404;
        }

        # [절대 삭제 금지] API 프록시 (백엔드 3001 포트로 전달)
        location /api/ {
            proxy_pass http://127.0.0.1:3001/api/;
            proxy_http_version 1.1;
            proxy_set_header Host \$host;
            proxy_set_header X-Real-IP \$remote_addr;
            proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto \$scheme;
        }

        # SPA 라우팅
        location / {
            try_files \$uri \$uri/ /index.html;
        }
    }
}
EOF'

sudo systemctl restart nginx
```

### 9.3 nginx 설정 변경 시 필수 체크리스트

| # | 체크 항목 | 누락 시 결과 |
|---|----------|-------------|
| 1 | `/api/` 프록시 블록 존재하는가? | **프론트엔드 전체 데이터 불러오기 불가** |
| 2 | `proxy_pass http://127.0.0.1:3001/api/;` 정확한가? | API 연결 실패 |
| 3 | 모든 `$`를 `\$`로 이스케이프했는가? | nginx 재시작 실패 |
| 4 | `index.html` 캐시 금지 블록 있는가? | 배포 후 변경사항 미반영 |
| 5 | `json/csv` 캐시 금지 블록 있는가? | 데이터 파일 변경 미반영 |
| 6 | `/assets/` 캐시 블록 있는가? | 빌드 파일 매번 재다운로드 |
| 7 | SPA `try_files` 블록이 마지막에 있는가? | 라우팅 오류 |
