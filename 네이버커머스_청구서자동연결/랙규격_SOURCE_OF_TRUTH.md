## ⛔ 핵심 주의사항 (스마트스토어 연동)

### 1. 파렛트랙 / 철판형: 치수 표기 방향이 다름
| | 스마트스토어 표기 예 | 실제 시스템 size 키 |
|---|---|---|
| 파렛트랙 일반 | (치수 순서 불명확) | `1390x1000` / `2590x1000` / `2790x1000` |
| 파렛트랙 철판형 | (치수 순서 불명확) | `1390x800` / `1390x1000` / `2590x800` / `2590x1000` |

- **800 깊이는 철판형에만 존재** — 일반 파렛트랙에는 없음
- **앞 숫자 = 로드빔 길이** (1390 / 2590 / 2790), **뒤 숫자 = 타이빔/깊이**

### 2. 하이랙 700kg는 존재하지 않음
- `colorLabelMap = { "700kg": "600kg" }` → 700kg은 표시용 레이블, 실제는 600kg
- **실제 color 키 없음** → 스마트스토어에서 700kg 표기가 오면 600kg로 처리해야 함

**옵션 선택 순서**: `version → weight → size → height → level → formType`

|---|---|
| **version** | `구형`, `신형` |
| **weight** | `2t`, `3t` |
| **size** (bom_data 기준) | `1390x1000`, `2590x1000`, `2790x1000` |

## 5. 하이랙

**옵션 선택 순서**: `color → size → height → level → formType`

### color (무게 포함) — data.json `색상` 배열 + 코드 로직 기준

| color 키 | 표시 무게 | 비고 |
|---|---|---|
| `메트그레이(볼트식)270kg` | 270kg | — |
| `블루(기둥)+오렌지(가로대)(볼트식)270kg` | 270kg | — |
| `메트그레이(볼트식)450kg` | 450kg | 45계열 size 없음 |
| `블루(기둥)+오렌지(가로대)(볼트식)450kg` | 450kg | 45계열 size 없음 |
| `블루(기둥.선반)+오렌지(빔)600kg` | 600kg | 80계열만 |

> ⛔ **700kg는 실제 존재하지 않음** → `colorLabelMap`에서 `"700kg":"600kg"` 매핑만 존재

### size — data.json 기준 (`{깊이}x{가로}` cm 단위)

| color 무게 | 유효한 size |
|---|---|
| **270kg** | `45x108`, `45x200`, `60x108`, `60x150`, `60x200` |
| **450kg** | `60x108`, `60x150`, `60x200` (45계열 전부 제외) |
| **600kg** | `80x108`, `80x150`, `80x200` |

> ⚠️ **270kg에는 `45x150`이 없고 `45x200`만 있음** (data.json 확인)  
> ⚠️ **EXTRA_OPTIONS 하이랙에 `45x150`이 있어** hide45=false일 때 추가됨 — 실제 매핑 시 주의  
> ⚠️ **`isHeaviest`(600/700kg) 조건에서 코드 라인 556이 80x200만 push** → `80x108`, `80x150`은 이미 rawSizes에서 나옴 (data.json에 존재)

### 하이랙 선반 개수/단 (calcHighRackShelfPerLevel)
- `d=108` → 1개/단
- `d=150` 또는 `d=200` → 2개/단

| 항목 | 유효한 값 |
|---|---|
| **height** | `150`, `200`, `250` (단위: cm, 고정 3가지) |
| **level** | `1단` ~ `6단` |
| **formType** | `독립형`, `연결형` |
