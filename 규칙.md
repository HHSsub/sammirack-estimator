# 시스템 규칙 및 ID 생성 가이드 (규칙.md)

> **[ABSOLUTE RULE]** 이 문서는 시스템의 **절대적 진실(Single Source of Truth)**입니다.
> 모든 코드 작성, 리팩토링, 기능 추가 시 이 문서에 정의된 **ID 규칙**과 **데이터 매핑**을 반드시 따라야 합니다.
> **절대** 임의로 ID 형식을 추론하거나 변경하지 마십시오.

---

## 1. 하이랙 (High Rack)

### 1-1. ID 구조

하이랙은 용도에 따라 두 가지 ID 체계를 가집니다.

| 용도 | 변수명 | 형식 (Format) | 비고 |
| :--- | :--- | :--- | :--- |
| **재고 관리** (Inventory) | `inventoryPartId` | `하이랙-{부품명}{색상}-사이즈{규격}{중량}kg` | **색상 포함**, **(볼트식) 포함** |
| **단가 관리** (Price) | `partId` | `하이랙-{부품명}-{규격}{중량}kg` | **색상 없음**, **중량 포함** |

### 1-2. 부품별 상세 규칙 (DB 기반)

#### A. 기둥 (Post) - **[Depth 통합 규칙 적용]**
*   **핵심 규칙**: 하이랙 기둥은 **깊이(Depth)**가 달라도(108, 150, 200 등) **너비(Width)와 높이(Height)**가 같으면 동일 부품으로 취급합니다.
*   **Inventory ID 예시**:
    *   `하이랙-기둥메트그레이(볼트식)270kg-사이즈60x높이150270kg` (⭕ 올바른 예 - `60x108` 아님)
    *   `하이랙-기둥블루(기둥)+오렌지(가로대)(볼트식)270kg-사이즈60x높이150270kg`
*   **Price ID 예시**:
    *   `하이랙-기둥-사이즈60x높이150270kg` (⭕ 올바른 예 - `60x108` 아님)
*   **주의사항**:
    *   기존의 `60x108`, `60x150`, `60x200` 등은 모두 **`60x`**로 통일됩니다.
    *   DB 마이그레이션을 통해 기존 데이터도 이 형식으로 통합되어야 합니다.
    *   재고 ID에는 `(볼트식)` 문구가 반드시 포함됩니다.

#### B. 선반 (Shelf)
*   **Inventory ID 예시**:
    *   `하이랙-선반메트그레이(볼트식)270kg-사이즈60x150270kg`
    *   `하이랙-선반블루(기둥)+오렌지(가로대)(볼트식)270kg-사이즈60x150270kg`
*   **Price ID 예시**:
    *   `하이랙-선반-사이즈60x150270kg` (색상X, 사이즈O, 중량O)
*   **주의사항**:
    *   단가 ID에도 `사이즈` 접두어가 붙습니다.

#### C. 로드빔 (Road Beam)
*   **Inventory ID 예시**:
    *   `하이랙-로드빔메트그레이(볼트식)270kg-150270kg`
*   **Price ID 예시**:
    *   `하이랙-로드빔-150270kg`
*   **주의사항**:
    *   로드빔 규격에는 `사이즈` 접두어가 붙지 않고 숫자만 옵니다 (예: `150`).

### 1-3. 아이보리 색상 규칙 (Special Case)
*   **Inventory ID**: `하이랙-{부품명}아이보리(볼트식){중량}kg-...`
    *   예: `하이랙-선반아이보리(볼트식)270kg-사이즈60x150270kg`
*   **Price ID**: 메트그레이/블루오렌지와 동일한 `partId` 공유 (색상 정보 제거)
    *   예: `하이랙-선반-사이즈60x150270kg`
*   **매핑 포인트**: 아이보리 색상을 선택해도 단가는 공통 단가표(`admin_prices`)의 ID를 사용해야 합니다.

## 2. 경량랙 (Light Rack)

### 2-1. ID 구조

경량랙은 재고/단가 ID가 **동일한 패턴**을 가지거나, 부품에 따라 다릅니다.

| 부품 | 형식 (Format) | 예시 |
| :--- | :--- | :--- |
| **기둥** | `경량랙-기둥{색상}-h{높이}` | `경량랙-기둥아이보리-h1800` |
| **선반** | `경량랙-선반{색상}-w{가로}xd{세로}` | `경량랙-선반아이보리-w900xd450` |
| **선반(단가)** | `경량랙-선반({가로})|사이즈 W{가로}xD{세로}` | `경량랙-선반(900)|사이즈 W900xD450` (주의: DB마다 다를 수 있음, 확인 필요) |
| **연결대** | `경량랙-연결대{색상}-w{규격}` | `경량랙-연결대아이보리-w900` |
| **받침(상/하)** | `경량랙-받침{상/하}{색상}-d{규격}` | `경량랙-받침상아이보리-d300` |

### 2-2. 특이사항
*   규격 표기에 소문자 `w`, `d`, `h`가 사용됩니다.
*   **색상**: `아이보리`, `블랙`, `실버` 등이 존재합니다.

## 3. 파렛트랙 (Pallet Rack)

### 3-1. ID 구조

파렛트랙은 '일반형'과 '철판형'으로 나뉩니다.

#### A. 파렛트랙 (일반/신형)
*   **기둥**: `파렛트랙-기둥{규격}-{규격}` (예: `파렛트랙-기둥2000-2000`, `파렛트랙-기둥h4500-높이h4500`)
*   **로드빔**: `파렛트랙-로드빔-{규격}` (예: `파렛트랙-로드빔-2590`)
*   **타이빔**: `파렛트랙-타이빔-{규격}` (예: `파렛트랙-타이빔-1000`)

#### B. 파렛트랙 철판형
*   **접두어**: `파렛트랙 철판형-` (띄어쓰기 주의)
*   **선반**: `파렛트랙 철판형-선반{규격}-사이즈{규격}x{깊이}`
    *   예: `파렛트랙 철판형-선반2590-사이즈2590x1000`

### 3-2. 3T 규격 (신형/옵션)
*   접미사 `_3t`가 붙는 경우가 있습니다.
    *   예: `파렛트랙신형-기둥-2000_3t`

---

## 4. 중량랙 (Medium Rack)

### 4-1. ID 구조
*   **기둥**: `중량랙-기둥-h{높이}` (예: `중량랙-기둥-h1800`) - **색상 없음**
*   **선반**: `중량랙-선반-w{가로}xd{세로}` (예: `중량랙-선반-w1500xd450`)
*   **받침(상/하)**: `중량랙-받침{상/하}-d{규격}` (예: `중량랙-받침상-d450`)

---

## 5. 스텐랙 (Stainless Rack)

### 5-1. ID 구조
*   **기둥**: `스텐랙-기둥-높이{높이}` (예: `스텐랙-기둥-높이180`)
*   **선반**: `스텐랙-선반-사이즈{가로}x{세로}` (예: `스텐랙-선반-사이즈50x120`)

---

## [개발자 필독] 수정 금지 및 주의사항

1.  **DB Key 불변의 법칙**: 코드로 ID를 생성할 때, 위 규칙과 DB(`admin_prices`, `inventory`)에 존재하는 포맷을 **절대적으로** 준수해야 합니다.
2.  **임의 데이터 추가 금지**: 없는 색상, 없는 규격을 코드로 강제 매핑하지 마십시오. (예: 아이보리 45x150 등)
3.  **검증 필수**: ID 생성 로직 수정 시, 반드시 실제 DB 덤프 데이터와 1:1 매칭 테스트를 수행하십시오.
